# 성능 최적화

## 모델 선택 전략

**Haiku 4.5** (Sonnet 대비 90% 성능, 3배 비용 절감):
- 빈번하게 호출되는 경량 agent
- 페어 프로그래밍 및 코드 생성
- 다중 agent 시스템의 워커 agent

**Sonnet 4.5** (최고의 코딩 모델):
- 주요 개발 작업
- 다중 agent 워크플로우 조율
- 복잡한 코딩 태스크

**Opus 4.5** (가장 깊은 추론):
- 복잡한 아키텍처 의사결정
- 최대 추론이 필요한 경우
- 연구 및 분석 태스크

## 컨텍스트 윈도우 관리

컨텍스트 윈도우의 마지막 20%에서는 다음을 피할 것:
- 대규모 리팩토링
- 여러 파일에 걸친 기능 구현
- 복잡한 상호작용 디버깅

낮은 컨텍스트 민감도 작업:
- 단일 파일 편집
- 독립적인 도구 생성
- 문서 업데이트
- 단순 버그 수정

## Ultrathink + Plan 모드

깊은 추론이 필요한 복잡한 태스크의 경우:
1. `ultrathink`로 사고력 강화
2. **Plan 모드**를 활성화하여 체계적 접근
3. 다회차 비평으로 "엔진 예열"
4. 역할별 하위 agent로 다각적 분석

## 빌드 트러블슈팅

빌드 실패 시:
1. **build-error-resolver** Agent 사용
2. 오류 메시지 분석
3. 점진적 수정
4. 수정할 때마다 검증
