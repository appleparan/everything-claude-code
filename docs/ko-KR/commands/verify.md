# 검증 명령어

현재 코드베이스 상태에 대해 포괄적인 검증을 수행합니다.

## 설명

다음 순서대로 정확하게 검증을 실행합니다:

1. **빌드 검사**
   - 이 프로젝트의 빌드 명령어 실행
   - 실패 시 오류를 보고하고 중지

2. **타입 검사**
   - TypeScript/타입 검사기 실행
   - 파일:줄번호를 포함한 모든 오류 보고

3. **Lint 검사**
   - 린터 실행
   - 경고 및 오류 보고

4. **테스트 스위트**
   - 모든 테스트 실행
   - 통과/실패 수 보고
   - 커버리지 비율 보고

5. **Console.log 감사**
   - 소스 파일에서 console.log 검색
   - 위치 보고

6. **Git 상태**
   - 커밋되지 않은 변경 사항 표시
   - 마지막 커밋 이후 수정된 파일 표시

## 출력

간결한 검증 보고서를 생성합니다:

```
검증: [통과/실패]

빌드:    [OK/실패]
타입:    [OK/X개 오류]
Lint:    [OK/X개 문제]
테스트:  [X/Y 통과, Z% 커버리지]
시크릿:  [OK/X개 발견]
로그:    [OK/X개 console.log]

PR 생성 준비: [예/아니오]
```

치명적인 문제가 있는 경우 해당 항목을 나열하고 수정 방법을 제안합니다.

## 매개변수

$ARGUMENTS 가능한 값:
- `quick` - 빌드 + 타입만 검사
- `full` - 모든 검사 (기본값)
- `pre-commit` - 커밋 관련 검사
- `pre-pr` - 전체 검사 + 보안 스캔
