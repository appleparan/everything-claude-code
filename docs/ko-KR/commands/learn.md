# /learn - 재사용 가능한 패턴 추출

현재 작업 세션을 분석하고 스킬로 저장할 가치가 있는 패턴을 추출합니다.

## 트리거

작업 세션 중 사소하지 않은 문제를 해결했을 때 언제든지 `/learn`을 실행하세요.

## 추출 대상

다음을 찾습니다:

1. **오류 해결 패턴**
   - 어떤 오류가 발생했나요?
   - 근본 원인은 무엇이었나요?
   - 무엇이 해결했나요?
   - 유사한 오류에 재사용할 수 있나요?

2. **디버깅 기법**
   - 직관적이지 않은 디버깅 단계
   - 효과적인 도구 조합
   - 진단 패턴

3. **우회 방법**
   - 라이브러리 특이점
   - API 제한 사항
   - 특정 버전에 대한 수정

4. **프로젝트 특정 패턴**
   - 발견된 코드베이스 관례
   - 내려진 아키텍처 결정
   - 통합 패턴

## 출력 형식

`~/.claude/skills/learned/[pattern-name].md`에 스킬 파일을 생성합니다:

```markdown
# [설명적인 패턴 이름]

**추출일:** [날짜]
**맥락:** [이 패턴이 적용되는 상황에 대한 간략한 설명]

## 문제
[이 패턴이 해결하는 문제 - 구체적으로 작성]

## 해결 방법
[패턴/기법/우회 방법]

## 예시
[해당되는 경우 코드 예시]

## 사용 시기
[트리거 조건 - 이 스킬을 활성화해야 하는 상황]
```

## 프로세스

1. 작업 세션에서 추출할 수 있는 패턴 검토
2. 가장 가치 있고 재사용 가능한 인사이트 식별
3. 스킬 파일 초안 작성
4. 저장 전 사용자에게 확인 요청
5. `~/.claude/skills/learned/`에 저장

## 주의 사항

- 사소한 수정은 추출하지 마세요 (오타, 간단한 구문 오류)
- 일회성 문제는 추출하지 마세요 (특정 API 중단 등)
- 향후 작업 세션에서 시간을 절약할 수 있는 패턴에 집중
- 스킬은 집중적으로 유지 - 스킬당 하나의 패턴
