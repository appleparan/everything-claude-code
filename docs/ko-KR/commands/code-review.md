# 코드 리뷰

커밋되지 않은 변경 사항에 대해 포괄적인 보안 및 품질 리뷰를 수행합니다:

1. 변경된 파일 가져오기: git diff --name-only HEAD

2. 각 변경된 파일에 대해 다음을 검사합니다:

**보안 문제 (치명적):**
- 하드코딩된 자격 증명, API 키, 토큰
- SQL 인젝션 취약점
- XSS 취약점
- 누락된 입력 유효성 검사
- 안전하지 않은 의존성
- 경로 탐색 위험

**코드 품질 (높음):**
- 50줄을 초과하는 함수
- 800줄을 초과하는 파일
- 4단계를 초과하는 중첩 깊이
- 누락된 오류 처리
- console.log 문
- TODO/FIXME 주석
- 공개 API에 누락된 JSDoc

**모범 사례 (중간):**
- 변이 패턴 (불변을 사용해야 함)
- 코드/주석에 이모지 사용
- 새 코드에 누락된 테스트
- 접근성 문제 (a11y)

3. 다음을 포함한 보고서 생성:
   - 심각도: 치명적, 높음, 중간, 낮음
   - 파일 위치 및 줄 번호
   - 문제 설명
   - 권장 수정 방법

4. 치명적 또는 높은 우선순위 문제가 발견되면 커밋 차단

보안 취약점이 있는 코드는 절대 승인하지 마세요!
